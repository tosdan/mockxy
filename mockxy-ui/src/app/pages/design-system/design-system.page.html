Design System

Il punto di riferimento per costruire l'app in modo coerente: una sola fonte per colori, tipografia e componenti, tutti basati sui design token del tema definitivo (aurora-2). Stack: design token (Tailwind @theme) + class-variance-authority + tailwind-merge + @ng-icons/lucide, con spartan-ng (brain) per i componenti con comportamento.

token in src/styles.css primitivi in src/app/ui/

Colori & token

Ogni colore e' un token semantico (custom property) esposto come utility Tailwind. Usa sempre i token, mai valori grezzi: cambiando un token, cambia in modo coerente tutta l'app. I valori qui sotto sono letti a runtime dal CSS, quindi restano sempre allineati.

{{ group.title }}

{{ group.desc }}

}

Button

Componente ad attributo <button ui-button> (cva + token). Il primario usa il gradiente brand; le altre varianti sono superfici/bordi dai token. 

<button ui-button variant="secondary" size="sm">Salva</button>

Switch

Costruito sul primitivo headless di spartan-ng (comportamento + accessibilita); lo stile arriva dai token. ON = --positive, OFF = grigio neutro, disabilitato = sbiadito. Taglie via input size: default e sm (liste/righe dense). 

<ui-switch [(checked)]="enabled" size="sm" ariaLabel="Abilita endpoint" />

Badge

Chip neutro + testo a tinta dal token. Un'unica resa per metodi HTTP, classi di status e tinte generiche tramite l'input tone. Due taglie via size: sm (default) e md (pill grande, es. metodo nell'header).

Input

Select

Select su @angular/cdk overlay (pattern combobox + listbox, ARIA + tastiera), coerente con Dialog e Dropdown menu. A differenza della select nativa, la tendina aperta e' completamente tematizzata coi token. API data-driven: [options] + [(value)]. Input tone (2xx/3xx/4xx/5xx) per la pill colorata con pallino (es. status code).

Code block

Blocco codice con numeri di riga ed evidenziazione JSON, sui token --json-* / --code.

Code editor

Editor di codice editabile basato su CodeMirror 6: evidenziazione della sintassi, numeri di riga, parentesi abbinate e ripiegamento, con il linguaggio scelto via [language] (json per i mock, javascript per handler/middleware). Per il JSON il lint è inline; la validazione che governa il salvataggio resta nel consumatore, che pilota [invalid] per il ring di errore. Usato per body/sorgente nella schermata Mocks.

{{ codeEditorInvalid() ? 'JSON non valido — il ring diventa rosso.' : 'JSON valido. Prova a romperlo per vedere lo stato invalid.' }} 

<ui-code-editor [value]="draft()" (valueChange)="draft.set($event)" [invalid]="invalid()" [minRows]="5" language="json" />
